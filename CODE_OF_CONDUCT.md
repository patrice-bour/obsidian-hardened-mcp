# Code of Conduct

This project adopts the
[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/),
by reference.

The full text is available at the link above. In short: contributors
and maintainers are expected to act with respect, openness, and good
faith in all project spaces (issues, pull requests, code review,
discussions).

## Reporting

To report a Code of Conduct concern, contact the maintainer at the
email address listed in [`pyproject.toml`](pyproject.toml) with the
subject prefix `[obsidian-full-mcp coc]`. Reports are handled
privately. For security vulnerabilities, see
[`SECURITY.md`](SECURITY.md) instead — that has its own private
channel.

## Scope

The Code of Conduct applies to all project-controlled spaces:

- This GitHub repository (issues, pull requests, discussions, the wiki)
- Direct correspondence about this project
- Public events where someone represents this project

It does **not** govern private interactions, third-party platforms
that host their own conduct rules, or unrelated activities by
contributors.
