# v0.1 follow-ups (deferred from code reviews)

This document collects findings from code reviews that were judged
non-critical for the milestone in progress and deliberately deferred.
Each entry must end up either implemented before the v0.1.0 tag (M8)
or explicitly closed with a rationale in the same document.

The convention used in entries:

- **Source** — review session that produced the finding (e.g. `M4-review`)
- **Severity** — Major / Minor / Strategic (Critical findings are fixed
  in-line and do not appear here)
- **Status** — `open` / `done` / `wontfix` (with rationale)
- **Refs** — file:line pointers when applicable

---

## From M4 review (b0b7862)

### M4-01 — `path_routing` built-in hook missing
- **Severity**: Major
- **Status**: open
- **Refs**: plan §4 "hooks built-in (iso_date, reserved_tags, path_routing)"
- The original plan called for three built-in hooks; M4 shipped two.
  `path_routing` (validate that `type:` matches the host folder, e.g.
  `offre-emploi → 01_Notes/Professionnel/Emploi/offres/`) needs a routing
  table format and a hook implementation. To revisit before v0.1.0 — most
  natural in M5 alongside any `routing.yaml` consumers.

### M4-02 — Plan promised "JSON Schema validation as baseline"; M4 made it opt-in
- **Severity**: Strategic
- **Status**: open
- **Refs**: plan §"Sécurité additionnelle"
- Decide before v0.1.0: keep opt-in (simpler, more honest about user
  responsibility) and amend the plan, or activate a default empty schema
  registry that runs even when the config file is absent.

### M4-03 — `_validate_kwargs` checks names but not value types
- **Severity**: Minor
- **Status**: open
- **Refs**: `validation/config_loader.py:164-174`
- `reserved_tags: forbidden: "not-a-list"` passes signature validation,
  ends up as `frozenset("not-a-list")` (per-character set). Add per-hook
  schema check, or move type validation into each hook's constructor.

### M4-04 — Schema-loading code not in dedicated module
- **Severity**: Minor
- **Status**: open
- **Refs**: `validation/config_loader.py:177-208`
- Plan called for `validation/schema_loader.py`. Inline (~30 lines) is
  acceptable for v0.1; extract when hot-reload arrives in v0.2+.

### M4-05 — TOCTOU symlink window in `_load_schemas`
- **Severity**: Minor
- **Status**: open (documented as out of scope)
- **Refs**: `validation/config_loader.py:190-203`, `docs/security-model.md`
- `resolve(strict=False)` → `relative_to` → `read_text` is a small
  TOCTOU window. Already covered by the "single-user trusted" assumption
  in the threat model; mention this specific case in the security doc
  for completeness.

### M4-06 — `_plain` silently coerces non-string dict keys to strings
- **Severity**: Minor
- **Status**: open
- **Refs**: `validation/config_loader.py:158`
- `{1: 2}` becomes `{"1": 2}` without warning. Reject explicitly and
  raise `ConfigError`.

### M4-07 — `to_plain_dict` does not flag bytes/set/Path
- **Severity**: Minor
- **Status**: open (impossible by current construction)
- **Refs**: `tools/_base.py:200-225`
- For v0.1 the parser refuses these on the read side and the writer
  refuses them too, so they cannot reach this function. Add a defensive
  `raise TypeError` for forward compatibility.

### M4-08 — Double-parsing of `new_content` in `_execute_write`
- **Severity**: Minor
- **Status**: open (perf)
- **Refs**: `tools/write.py:55-85`
- Validation parses, then atomic write may re-parse downstream. Profile
  on a large vault; cache the parse result if measurable.

### M4-09 — `_BUILTIN_HOOKS: dict[str, Any]` could be `dict[str, type[ValidationHook]]`
- **Severity**: Minor (cosmetic)
- **Status**: open
- **Refs**: `validation/config_loader.py:55`
- A `cast(...)` at the lookup site lets the registry type be precise.

### M4-10 — `_Warning` / `_Rejection` private names leak into public `HookReport`
- **Severity**: Minor
- **Status**: open
- **Refs**: `validation/hooks.py:96-113`
- Rename to `HookWarning` / `HookRejection`, expose them in
  `validation.hooks.__all__`.

### M4-11 — Missing tests for config-loader error branches
- **Severity**: Minor
- **Status**: open
- **Refs**: `validation/config_loader.py` lines 76, 87, 96, 109, 117,
  147, 150, 181, 187, 204-205
- ~15% uncovered, all user-facing error paths. Add 4-5 tests:
  config absent (already covered indirectly), config empty, hooks not
  a list, JSON Schema malformed at load.

### M4-12 — `BaseException` (KeyboardInterrupt, SystemExit) escapes the hook catch
- **Severity**: Minor (documentation)
- **Status**: open
- **Refs**: `validation/hooks.py:155`
- Document in `docs/config-reference.md` that a hook can kill the server
  via `SystemExit` — choose hooks you trust.

### M4-13 — No config hot-reload
- **Severity**: Strategic
- **Status**: open (acceptable v0.1)
- **Refs**: `validation/config_loader.py`
- Editing `.obsidian-power-mcp.yaml` requires a server restart. Note in
  config-reference.md (already documented) and consider a `reload`
  meta-tool in v0.2.

### M4-14 — ConfigError surfaces as raw traceback at startup
- **Severity**: Minor (UX)
- **Status**: open
- **Refs**: `__main__.py`
- Wrap `create_server(config)` in `__main__.main` so a `ConfigError`
  prints a clean message to stderr and exits with code 3 (config error).

## From M5 review (search + wikilinks)

### M5-01 — `search_notes` engine: no ripgrep
- **Severity**: Strategic
- **Status**: open
- **Refs**: plan §M5 "ripgrep + fallback Python"; `tools/search.py:131`
- v0.1 ships pure-Python search. The result envelope already carries
  `engine: "python"` so a future ripgrep mode is a transparent upgrade.
  Target ≤ 300 ms on the pbkm vault (3618 notes); current Python likely
  hits several seconds. Implement before v0.1.0 if benchmark confirms.

### M5-02 — `search_notes` and `resolve_wikilink` lack a TTL cache
- **Severity**: Strategic
- **Status**: open
- **Refs**: plan §M5 "index lazy en mémoire avec TTL 60s"
- Each call rescans the whole vault. Acceptable for a single-user local
  server; a DoS vector if exposed to remote callers. Cache the
  `iter_markdown` enumeration + parsed frontmatters with a 60 s TTL.

### M5-03 — Snippet length: chars vs bytes mismatch
- **Severity**: Minor (cosmetic)
- **Status**: open
- **Refs**: `tools/search.py:35` (`_SNIPPET_MAX_BYTES`)
- Constant is named `_BYTES` but slicing `[:200]` is in chars. Either
  rename to `_SNIPPET_MAX_CHARS` (cheapest) or do a real byte-bounded
  slice via UTF-8 round-trip with `errors="ignore"`.

### M5-04 — `_walk_for_match` only matches strings
- **Severity**: Minor
- **Status**: open
- **Refs**: `tools/search.py:228-242`
- `count: 42` in frontmatter and query "42" → no match. Either document
  ("frontmatter mode matches strings only") or stringify scalars
  (`isinstance(node, (int, float, bool))` → coerce).

### M5-05 — `md_path_to_vault_path` typed via `# type: ignore`
- **Severity**: Minor (quality)
- **Status**: open
- **Refs**: `tools/search.py:143`
- Add `from pathlib import Path` and `md_path: Path` annotation, drop
  the `type: ignore`. Rename to `_md_path_to_vault_path` (private) since
  it's only used internally.

### M5-06 — Tests missing for error paths in `search_notes`
- **Severity**: Minor
- **Status**: open
- **Refs**: `tools/search.py:87, 96`
- The `skipped_parse` counter is now tested (M5 review fix); add tests
  for `skipped_read` triggered by an oversized file, an `.icloud`
  placeholder, or a permission-denied file.

### M5-07 — `_walk_for_match` recursion not bounded
- **Severity**: Minor (defensive)
- **Status**: open
- **Refs**: `tools/search.py:228`
- Frontmatter parser caps depth via `_ensure_safe_value` on the WRITE
  path, but a hand-edited file could carry 10k levels. Add a
  `max_depth=32` guard analogous to `_MAX_VALUE_DEPTH`.

### M5-08 — `_candidates_by_basename` case-insensitive on all platforms
- **Severity**: Minor (documentation)
- **Status**: open
- **Refs**: `tools/wikilink.py:170-178`
- Compares names lowercased, so `Alpha.md` matches a query for `alpha`
  on every filesystem. APFS-friendly, but a surprise on case-sensitive
  ext4. Document the choice in the docstring.

### M5-09 — `resolve_wikilink` rescans on every call
- **Severity**: Minor (perf, also covered by M5-02)
- **Status**: open
- **Refs**: `tools/wikilink.py:172-180`
- Same root cause as M5-02; will be fixed by the index cache.
